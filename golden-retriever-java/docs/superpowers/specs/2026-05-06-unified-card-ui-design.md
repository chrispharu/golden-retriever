# 設計文件：統一卡片式資產追蹤介面與精度優化

## 1. 專案背景
當前介面存在資訊密度過高、字體偏小、輸入精度不足（不支援小數點）以及部分 UI 元素（如日期選擇器）在小螢幕或特定容器下會被裁切的問題。為了提升使用者體驗與量化分析的直觀度，本設計將參考 `stock_tracker_mockup.html` 進行全面重構。

## 2. 核心目標
- **統一風格**：將觀察清單、庫存、診斷結果整合為一致的「卡片式」設計。
- **直觀監測**：新增頂部總覽欄、52 週高低點進度條及量能狀態標籤。
- **精度優化**：全面支援股數與價格的小數點輸入與顯示（適用於定期定額）。
- **佈局修復**：解決新增庫存時日期選擇器被裁切的問題。

## 3. 功能規格

### 3.1 頂部總覽欄 (Summary Bar)
- **總市值**：顯示所有持股換算為台幣後的總額。
- **今日損益**：顯示今日漲跌金額與百分比（綠漲紅跌）。
- **投入成本**：顯示累計投入的台幣總額。
- **持倉/現金比**：顯示資產分配比例（需從後端配置讀取現金餘額）。

### 3.2 個股卡片設計 (Stock Card)
- **視覺定錨**：左側加入 4px 邊條（上漲=綠色，下跌=紅色）。
- **狀態標籤**：顯示「盤中開市」或「已休市」。
- **核心數據**：
    - 名稱/代碼/現價/漲跌幅（大字顯示）。
    - 持股市值、成本均價、累積報酬率（三欄佈局）。
- **進度條**：顯示目前股價在「52 週高低點」之間的相對位置。
- **量能標籤**：根據量價關係顯示「量增價漲」、「量縮價跌」等警示標籤。

### 3.3 數據精度調整
- **Frontend**: 
    - `PortfolioItem` 與 `InventoryRecord` 介面中的 `price` 與 `shares` 型別維持 `number`，但顯示格式改為 `1.2-4`。
    - `input[type="number"]` 增加 `step="any"`。
- **Backend**: 
    - 確保 `InventoryEntity` 欄位與資料庫 `BigDecimal(18,4)` 對齊。

### 3.4 交互優化
- **新增庫存表單**：從「行內展開」改為「**中央彈出式對話框 (Modal)**」，提供充足空間顯示 `input[type="date"]` 及其下拉選單。

## 4. 技術架構
- **元件重構**：
    - 修改 `sidebar.html/scss` (觀察清單)。
    - 修改 `inventory.html/scss` (庫存清單)。
- **樣式標準**：
    - 採用 Vanilla CSS/SCSS。
    - 遵守資安規範，禁止行內樣式。
    - 使用 CSS Variables 定義品牌色（Friendly Green, Playful Red, Soft Cream）。

## 5. 驗證標準
1.  **介面自適應**：卡片在 300px ~ 1200px 寬度下皆能正確排列且不重疊。
2.  **小數點測試**：輸入 `12.3456` 股與 `123.45` 價格，存入資料庫後重新讀取需保持一致。
3.  **日期選擇器**：在各個解析度下開啟「新增庫存」，日期選擇器必須完全可見。
4.  **連動性**：修改庫存後，頂部總覽欄應自動更新數值。
